Introduction¶

Overview¶

With the Hotel Bookings Reporting application it is possible to explore hotel booking data and provide reports on upcoming bookings.

Prerequisite¶

Python3 is a requirement for running the Hotel Bookings Reporting application. If you don’t already have Python3 installed, you can download it from here

Installation¶

  1. Clone git repository

git clone https://github.com/aydtmiri/hotel-bookings-analysis.git

  1. Install the project requirements by running following commands in the terminal:

cd .\hotel-bookings-analysis\

pip install -r requirements.txt

Start application¶

Note

If you have not finished the installation yet, please do so before proceeding.

In order to run the application, please run following commands in the terminal:

cd .\hotel-bookings-analysis\

py  .\hotel_booking_app\view\gui.py

User Interface¶

After running the application a window will pop up, including following input fields:

  • A date entry field : Enter a date in YYYY-MM-DD format, to search for all active bookings at that date. It is additionally used to calculate the total number of guests expected to be in residence per day over the next 7 days.

  • CSV Upload button: Upload the CSV file, on which the analysis should be performed on.

How to run an analysis¶

Note

If you don’t know how to run the application yet, please refer to the Start application section.

To perform the analysis, please provide a date and a CSV file.

Note

The CSV file, containing the relevant dataset, can be downloaded here.

After pressing the button Analyse Data, two result tables will be presented.

1. Currently active bookings: This table contains all bookings, that are currently active or will begin within the next 7 days.

2. Total guests per day: This table shows the total number of adults, children and, babies expected to be in residence per day over the next 7 days.

Both tables are also saved as CSV files and can be found under following path:

cd \hotel-bookings-analysis\data\results

In the folder data the raw and resulting CSV files are located.

  • data\raw : Contains uploaded CSV files.

  • data\results : Contains resulting CSV files.

The folder docs contains the documentation of the application.

In the package hotel_booking_app all the source code is located.

  • hotel_booking_app\settings.py : Here, the paths of the folders are defined to which the CSV files are saved to.

All the logic of the application is located in the package hotel_booking_app\src. It contains two packages;

  • hotel_booking_app\preparation, containing logic to prepare data for further use, and

  • hotel_booking_app\processing, where all files are located in which data is processed.

    • The file analyse_data.py contains methods for analysing hotel bookings.

The hotel_booking_app\view package contains the GUI of the application.

The entry point of the application is the file hotel_booking_app\view\gui.py

Functionality¶

In the following the base functionalities are explained.

Clean data¶

The function import_clean_data(csv_file_path, csv_file_save_to_path,threshold) reads a CSV file, cleans it and saves the result to data\raw. It drops all rows that have more than a certain percentage of NaN entries. This can be controlled by the threshold input.

Analyse active bookings¶

With the help of the function get_active_bookings(csv_file_path, selected_date, time_span) it is possible to get all bookings, that are currently active or will begin in the next x days. The result is saved as a pandas data frame, which is then cleaned and saved as CSV file by the function analyse_active_bookings(csv_file_path, csv_file_save_to_path, selected_date, time_span) . The following UML diagram gives an overview of the procedure.

In order to receive those bookings, the arrival and leaving dates are identified. In a next step, all bookings where the arrival date is within the next x days, and the leaving date is after the current date are selected. Next, the analysis columns are dropped and the result data frame is saved to a CSV file.

Total guest per day¶

The function analyse_total_guests(csv_file_path, csv_file_save_to_path, selected_date,time_span) determines the total number of adults, children, and babies over the next x days. The following UML diagram gives an overview of the function.

The analysis is based on the resulting pandas data frame of the function get_active_bookings(csv_file_path, selected_date,time_span).

In a first step a new pandas data frame is created which contains the range between the arrival date and the leaving date. Next, this data frame is merged with the columns for adults, children, and babies. The result is a data frame in which every booking is assigned to the corresponding date of stay.

Lastly, by grouping by the stayed dates and sum up the number of adults, children, and babies, the total number of guest in the next x days is calculated. The resulting data frame is saved to a CSV file.
